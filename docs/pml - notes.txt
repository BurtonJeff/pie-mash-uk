In the Supabase dashboard, go to Authentication → Email Templates.

  You'll find templates for:
  - Confirm signup — the email confirmation link
  - Invite user
  - Magic link
  - Change email address
  - Reset password

  Each template lets you edit the Subject and Body (HTML). You can use these variables:

  ┌────────────────────────┬─────────────────────────────────────────────────┐
  │        Variable        │                      Value                      │
  ├────────────────────────┼─────────────────────────────────────────────────┤
  │ {{ .ConfirmationURL }} │ The confirmation/action link                    │
  ├────────────────────────┼─────────────────────────────────────────────────┤
  │ {{ .Email }}           │ The user's email address                        │
  ├────────────────────────┼─────────────────────────────────────────────────┤
  │ {{ .Token }}           │ The OTP token (if using OTP flow)               │
  ├────────────────────────┼─────────────────────────────────────────────────┤
  │ {{ .SiteURL }}         │ Your site URL (set in Auth → URL Configuration) │
  └────────────────────────┴─────────────────────────────────────────────────┘

  For example, a minimal confirmation body:
  <h2>Welcome to Pie & Mash</h2>
  <p>Click below to confirm your email address:</p>
  <p><a href="{{ .ConfirmationURL }}">Confirm email</a></p>

  To use a custom SMTP server (so emails come from your own domain like noreply@pml.org.uk):
  - Go to Project Settings → Auth
  - Scroll to SMTP Settings
  - Enable custom SMTP and enter your provider's credentials

  The free Supabase SMTP has a 2 emails/hour rate limit, so for production you'll want a custom SMTP provider (Resend, SendGrid, Postmark, etc.).
